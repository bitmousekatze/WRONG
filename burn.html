// Questions

Burn FAQ

Do I need to add a memo when sending?

+

No. Each event has its own dedicated burn address — the address itself tells the ledger which event you're burning for. Just send $WRONG to the event's address and you're done. No memo, no reference code, nothing extra.

Where do I find the burn address for a specific event?

+

On the Burn Ledger — click any event card to expand it. The unique burn address for that event is shown inside the card with a copy button. Each event's address is different, so make sure you're copying from the right card.

Can I burn for multiple events?

+

Yes — send separate transactions to each event's burn address. Each one is tracked independently on the ledger. No limit on how many events you burn for.

Can I burn tokens I bought before launch?

+

Yes — any $WRONG in your wallet can be burned at any time. Retroactive events (Mar 24 – Apr 6) are open for community burns now. Pick any of them from the ledger and burn toward their target.

Is the burn truly permanent?

+

Yes. Each event's burn wallet is a Solana address with no private key — tokens sent there cannot be moved by anyone. Fully public and verifiable on Solscan. Every transaction in is permanent and on-chain forever.

Does my wallet address get exposed publicly?

+

Yes — your Solana wallet address is visible on the ledger next to your burn amount. Solana transactions are public by design. Your wallet address is not your identity, but if you've connected it to other platforms it may be traceable. Use a fresh wallet if anonymity matters to you.
